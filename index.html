¿Que es una base de datos, cual es la estructura y que es un modelo entidad-relacion?

¿Qué es una Base de Datos?

Una base de datos es un conjunto de datos organizados y estructurados que se almacenan en un sistema informático. Estas bases de datos permiten la gestión, el almacenamiento y la recuperación de información de manera eficiente.

Modelo Entidad-Relación (ER)

El modelo entidad-relación es una herramienta utilizada en el diseño de bases de datos. Este modelo permite representar las entidades (objetos o conceptos del mundo real) y las relaciones entre ellas. Las entidades se representan como tablas en la base de datos, y las relaciones se definen mediante claves primarias y foráneas.

Estructura de una Base de Datos

La estructura de una base de datos se compone de tablas, columnas y filas. Cada tabla representa una entidad, las columnas representan los atributos de la entidad, y las filas representan los registros o instancias de la entidad. Además, las bases de datos suelen incluir índices, vistas y procedimientos almacenados para optimizar el acceso a los datos.

1.Tablas

Definición: Son la base de la base de datos, donde se almacenan los datos en filas y columnas.

  • Filas: Cada fila representa un registro único.
  • Columnas: Cada columna representa un atributo del registro.

2.Campos

Definición: Son las columnas de las tablas que contienen datos específicos.

  • Tipos de Datos: Pueden ser numéricos, texto, fecha, booleanos, etc.

3.Claves primarias

Definición: Un campo (o conjunto de campos) que identifica de manera única cada registro en una tabla.

  • Ejemplo: Un ID de usuario.

4.Claves Foraneas

Definición: Un campo que establece una relación entre dos tablas, apuntando a la clave primaria de otra tabla.

  • Funcion: Mantiene la integridad referencial entre tablas.

5.Indices

Definición: Estructuras que mejoran la velocidad de las consultas en la base de datos.

  • Uso:Permiten acceder a los datos más rápidamente.

6.Vistas

Definición: Consultas almacenadas que presentan datos de una o más tablas.

  • Funcion: Simplifican el acceso a datos complejos.

7.procedimientos almacenados

Definición:Conjuntos de instrucciones SQL que se almacenan en la base de datos.

  • Uso: Automatizan tareas y mejoran la eficiencia.

8.Triggers

Definición: Procedimientos que se ejecutan automáticamente en respuesta a ciertos eventos en la base de datos (como inserciones, actualizaciones o eliminaciones).

  • Funcion:Mantienen la integridad de los datos y realizan auditorías.

9.Esquemas

Definición: Estructuras que organizan las tablas y otros objetos dentro de la base de datos.

  • Uso: Facilitan la gestión y el acceso a los datos.

10.Relaciones

Definición: Conexiones entre tablas que definen cómo los datos de una tabla están relacionados con los de otra.
